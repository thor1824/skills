## Agent Brief

**Issue category:** enhancement
**Summary:** Establish the first runnable Team Lunch Picker slice with seeded **Team members**, a reusable **Restaurant** catalog, acting **Team member** selection, and a unique current-day **Lunch poll** that can be viewed or created.

**Current behavior:**
The repo is still a Vite React TypeScript starter. There is no Team Lunch Picker domain model, no seeded **Team members** or **Restaurants**, no notion of an acting **Team member**, and no **Lunch poll** creation or display behavior.

**Desired behavior:**
The application should boot into a recognizable Team Lunch Picker flow where seeded **Team members** and reusable **Restaurants** are available, the user can explicitly choose the acting **Team member**, and the app can show the current day's **Lunch poll** if it exists or create it if it does not. There must be at most one **Lunch poll** for a given day, and the poll must display a fixed set of **Restaurants** end-to-end in the UI.

**Key interfaces:**
- **Lunch poll** creation workflow — should enforce uniqueness per day and produce a poll with a fixed set of reusable **Restaurants**
- Acting **Team member** selection — should be explicit in the app without introducing authentication concepts
- Seeded **Team member** and **Restaurant** sources — should provide a small known dataset the app can load immediately
- Current-day **Lunch poll** read model — should support "show existing or create new" behavior from the user-facing app

**Acceptance criteria:**
- [ ] The app loads with seeded **Team members** and seeded reusable **Restaurants**
- [ ] The user can select the acting **Team member** in the app without authentication
- [ ] The app can show the current day's **Lunch poll** if it exists, or create it from the reusable **Restaurant** catalog if it does not
- [ ] The app prevents creation of more than one **Lunch poll** for the same day
- [ ] The current day's **Lunch poll** displays its fixed list of **Restaurants** end-to-end in the UI

**Out of scope:**
- Voting behavior
- Weighted scoring and **Winner** derivation
- Poll closure and closed-poll history
- Full persistence hardening beyond what this slice needs to run end-to-end
